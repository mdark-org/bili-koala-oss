### Stop-Slow

Stop-Slow 是一个开元的写作技能工具，专门帮 LLM 识别并去掉那些一眼就能看出的 AI 生成的文字痕迹。它把 AI 写作的套路归成几类：轻嗓子式的开场白、商业黑话、空洞的断言、过度的评论以及滥用破折号等毛病，并给出明确的进阶清单。它还配了一套质量打分系统，从直接性、节奏、可信度、真实感、信息密度五个维度按 1 到 10 分打分。50 分里，低于 35 分就要求重写。它以 skill 的形式分发，能接入任意 agent。

### MQ

MQ 是一个用 Rust 写的命令行工具，把处理 JSON 的 JQ 那套思路搬到了 Markdown 上，用途是查询、过滤、转换 Markdown。你可以用类似 JQ 的语法从文挡里抽取指定章节、提取所有标题或代码块，批量改写多个文挡，再把结果导出成 JSON、CSV、YAML、HTML 等格式。它针对 LLM 工作流做了设计，方便把 Markdown 内容切片，喂给大模型或处理模型输出。Koala 认为 MQ 把结构化查询能力带给纯文本格式，在 LLM 时代尤其实用，当大量内容需要被切分，过滤后送进模型，一个可组合、可脚本化的 Markdown 处理其会比手写解析高效得多。

### The Website Specification

The Website Specification 是一份开元的网站技术标准，把一个像样的网站应该具备的特性归纳成十大类，涵盖基础结构、SEO、无障碍、安全、性能、隐私、国际化，以及一个新颖的类别：Agent 可读性。它不绑定技术栈，无论你用 WordPress、Next.js 还是纯静态 HTML 都适用，还提供交互式审计清单。Koala 认为这份规范的价值在于把碎片化的最佳实践系统化，并坚持用标准做备书，可信度比一般的经验清单更高。其中专门列出 Agent 可读性这个类别，反映了网站的读者不再只是人类，还有越来越多的 AI 智能体。能不能被 Agent 正确理解和调用，正成为网站建设的新考量。

### GitHub Copilot CLI

GitHub Copilot CLI 此更新带来了几个有意思的能力。最受关注的是 `gh copilot alias`，小黄鸭功能等就是可用。它是一个内质的批评式 Agent，会主动审视你的方案、设计、实现和测试，专挑漏洞和忙点。语音输入和定时任务等实用功能也正式上线，终端界面也重新设计，更方便浏览 Issue 和 PR，多种配色和屏幕阅读器。Koala 认为 GitHub 在 Coding Agent 之战中还没投降，投入 Copilot CLI 而减少 VS Code 上的投入，也是顺应潮流之举。虽然近期对 GitHub AI 方面产品功能的批评声音不少，但这个世界最大的开发者社区，仍然是有力的竞争者。

### Hocus-Pocus

Hocus-Pocus 是一个基于 Yjs 的寄插即用的写作后端，用 Web Socket 把多人实时的编辑的同步逻辑封装好。它依托 CRDT，这种无冲突复制数据结构，让多个用户同时改一份文挡时自动合并，不会互相覆盖。开发者无需自己处理复杂的同步算法，架构高度可扩展，比如可以接 SQLite 做持久化，也提供云托管服务。TipTap Collab 也支撑了 TipTap 编辑器的协同编辑功能。Koala 认为在线写作几乎成了文挡、白板、笔记类产品的标配，但自己从零实现 OT 或 CRDT 同步门槛很高。Hocus-Pocus 把这块基础设施做成开箱即用的后端，和富文本编辑器 TipTap 同源，生态衔接顺滑。

### DriftVM

DriftVM 是一个实验性的 Vland 合成器，用无限画布的思路重新想像窗口管理。所有窗口以原始大小散落在一张无边界的二维画布上，屏幕则像一台相机，你通过平移和缩放，在画布里游走，窗口靠近时会自动吸附成组，可以整体移动和缩放。它支持触摸板原生关心的惯性滑动和捏合缩放等手势操作，对多显示器支持良好，还能通过 Unix Socket 脚本化控制。Koala 认为主流窗口管理无非平铺和堆叠两条路，DriftVM 把白板类应用流行的无限画布概念搬到了桌面管理层，是个少见的方向。它契合触摸板时代的交互直觉，使用 Linux 桌面的极客们可以尝鲜。
